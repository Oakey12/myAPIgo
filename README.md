Notes API (myAPIGo)

RESTful API сервис на языке Go, предназначенный для управления заметками. Архитектура приложения спроектирована в соответствии со стандартами корпоративной бэкенд-разработки на Go: используется многослойная структура каталогов, потокобезопасные механизмы хранения данных в оперативной памяти, кастомные промежуточные слои (middleware) для сквозного логирования, строгая валидация входящих запросов и автоматическая спецификация эндпоинтов через Swagger UI.

Технические характеристики и особенности архитектуры

Разделение ответственности (Clean Architecture): Код организован по стандартному шаблону проектирования Standard Go Project Layout (cmd, internal/handlers, internal/structs, utils). Каждый пакет решает строго изолированный набор задач (сетевой транспорт, бизнес-логика, инфраструктура).

Потокобезопасность (Thread-Safety): Доступ к структурам данных в памяти контролируется с помощью примитивов синхронизации sync.RWMutex. Это предотвращает состояние гонки (race conditions) при одновременной обработке конкурирующих запросов от множества клиентов.

Семантически верное обновление ресурсов:

Метод PUT осуществляет полную замену ресурса в соответствии со спецификацией HTTP.

Метод PATCH реализует частичную модификацию полей. Применение указателей (*string) в структурах запросов позволяет отличать ситуации, когда поле было намеренно передано пустым, от случаев, когда оно отсутствовало в теле JSON.

Многоуровневая валидация входящих данных:

Первичная валидация типов данных и структуры документа выполняется на этапе декодирования силами стандартного парсера encoding/json.

Вторичная (бизнес-валидация) контролирует отсутствие пустых значений, ограничения длины строк и корректность числовых идентификаторов.

Автоматическое документирование: Спецификация OpenAPI 2.0 генерируется на основе декларативных аннотаций к обработчикам при помощи утилиты swag.

Логирование запросов: Промежуточный слой регистрирует параметры каждого входящего запроса (HTTP-метод, URI, статус ответа и точное время выполнения).

Технологический стек

Язык программирования: Go (Golang) версии 1.22 или выше.

HTTP-маршрутизатор: Стандартный мультиплексор net/http с поддержкой расширенной маршрутизации.

Инструменты документирования: OpenAPI 2.0 / Swagger UI (github.com/swaggo/swag, github.com/swaggo/http-swagger/v2).

Структура каталогов проекта

Проект имеет следующую иерархическую структуру:

myAPIGo/
├── cmd/
│   └── main.go         # Точка входа: сборка зависимостей, настройка маршрутизатора и запуск сервера
├── internal/
│   ├── handlers/       # Транспортный уровень (обработка HTTP, декодирование и валидация данных)
│   └── structs/        # Доменный уровень (модели данных Note и потокобезопасное хранилище NoteStore)
├── utils/
│   └── logger.go       # Инфраструктурный уровень (middleware для логирования HTTP-трафика)
├── docs/               # Статические файлы Swagger-спецификации (генерируются автоматически)
├── go.mod              # Файл конфигурации модулей и зависимостей Go
└── testing.http        # Набор сценариев интеграционного тестирования

Инструкция по установке и запуску

1. Клонирование репозитория

Склонируйте исходный код проекта и перейдите в его корневую директорию:

git clone [https://github.com/Oakey12/myAPIGo.git](https://github.com/Oakey12/myAPIGo.git)
cd myAPIGo


2. Загрузка зависимостей

Выполните синхронизацию и очистку модулей зависимостей Go:

go mod tidy


3. Генерация Swagger-документации

При внесении изменений в аннотации обработчиков или структуры данных необходимо обновить статические файлы спецификации:

# Установка утилиты генерации в операционную систему
go install [github.com/swaggo/swag/cmd/swag@latest](https://github.com/swaggo/swag/cmd/swag@latest)

# Запуск процесса сборки документации
swag init -g cmd/main.go


4. Запуск приложения

Скомпилируйте и запустите сервер:

go run cmd/main.go


Успешный запуск сопровождается следующими записями в системном выводе:

2026/05/18 22:30:00 Server running on http://localhost:8012
2026/05/18 22:30:00 Swagger documentation available on http://localhost:8012/swagger/index.html


Инструменты тестирования

Интерактивное тестирование через Swagger UI

После старта приложения спецификация доступна в браузере по адресу:
http://localhost:8012/swagger/index.html

Используйте встроенную функцию "Try it out" в каждом интерактивном блоке метода для генерации реальных запросов к работающему экземпляру сервера.

Локальное тестирование средствами IDE

В корневом каталоге проекта находится файл testing.http. Разработчики, использующие встроенные HTTP-клиенты в GoLand или VS Code, могут последовательно запускать запросы для проверки жизненного цикла заметок непосредственно из интерфейса редактора кода.
